Cuemath Content Pages — Revamp Scope & Roadmap

URL structure, phase breakdown, and decisions needed for the curriculum pages revamp. Primary goal: US content growth, driving leads through free curriculum access.

Owner: Manasa (US Content Growth) Last updated: June 2026 Status: Roadmap planning

Purpose

Bring the complete Cuemath curriculum (currently only accessible in the Leap app) to the public website — free with login. The curriculum content is the lead magnet; 1:1 tutoring remains the paid product.

Primary goal Drive leads from parents discovering Cuemath through curriculum browsing — same way IXL and Khan Academy convert. 200 users/month visit cuemath.com/curriculum today with near-zero signups. IXL gets ~10k/month on their curriculum pages.

Content source

All curriculum content (chapters, lessons, lesson objectives, worksheets, exercises) comes from Leap — the internal tool the curriculum team already maintains. The website renders what Leap already has. No new content pipeline needed.

Key constraint Phase 1 is content discovery only — no lesson player, no login required. The page must answer: "Does Cuemath cover what my child is learning in school?" If yes, the CTA converts.

URL Structure

All pages are geo-specific in feel (board/standard filter) but serve the same underlying US curriculum for Phase 1. Separate URLs per geo come in Phase 2+.

URL decisions needed

Should grade pages be geo-specific in URL?
Recommendation: No for Phase 1. Single URL per grade, geo auto-detected and shown as a dropdown filter. Separate URLs (e.g. /texas-grade-3-math/) are a Phase 2 SEO play once we have enough content volume.
cuemath.com/math/ or cuemath.com/curriculum/?
/math/ is the right URL. /curriculum/ is what exists today and has near-zero authority. /math/ is shorter, cleaner, and matches how IXL (ixl.com/maths) and Khan (khanacademy.org/math) structure it. Redirect /curriculum/ → /math/.
K-8 or K-12 for Phase 1?
K-8 for Phase 1 (matching current Cuemath product). Grades 9-12 come in Phase 2 with test prep angle (SAT, board exams). Don't launch empty pages.
Login required to view curriculum?
No login for Phase 1 browsing. Login required to: save progress, download worksheets, access lesson player. This is the IXL and Khan model — content is discovery, login is retention.

Page Specs — Phase 1

1. Homepage — cuemath.com/math/

  • Hero: Headline + grade/board selector card. No booking form in hero. CTA is "Explore Grade X →" leading to grade page.
  • Grade cards (K–8): Each shows grade name, age range, 3 topic areas, chapter count, lesson count. All data from Leap.
  • Board filter: Common Core (US) / CBSE / ICSE / UK / Singapore. Auto-detects by geo. Updates grade card descriptions.
  • Trust signals: Compact bar — standards aligned, student count, rating.
  • Features strip: What makes Cuemath different from IXL/Khan (live tutoring angle).
  • SEO content: What is K-8 math curriculum, FAQs, grade grid linking all grade pages.

2. Grade Page — cuemath.com/math/grade-3-math/

  • Sidebar (sticky): Grade pills (switch grade), board/geo dropdown, chapter nav with lesson counts, booking CTA.
  • Hero: "Grade 3 Math Curriculum · Common Core" eyebrow. Stats: X chapters, X lessons, standards, age range.
  • Standards strip: CCSS domains covered (3.OA, 3.NF, 3.MD etc).
  • Chapter cards (expandable): Chapter title → expands to lesson list. Each lesson shows: lesson name, skill count, "Key" / "New" tags.
  • Lesson objectives: First lesson in each chapter shows objectives pulled from Leap ("By the end of this lesson, I will be able to: …"). This is the differentiator vs IXL.
  • Per-chapter CTA: "Get help with [Chapter name] →" — contextual booking trigger.
  • SEO section: What Grade 3 covers, Common Core alignment, FAQ (4–6 questions), grade grid.

Open Questions for Roadmap Meeting

  1. Leap API: Is there an existing API to pull chapter/lesson data from Leap, or does this need to be built? Who owns this on the engineering side?
  2. Content ownership: When curriculum team updates Leap, does the website update automatically, or is there a manual sync step?
  3. Login infrastructure: Does the website have a login system that Phase 2 progress-saving can hook into, or does this need to be built from scratch?
  4. Redirect: When does /curriculum/ redirect to /math/? Before Phase 1 launch or after?
  5. Boards in scope for Phase 1: Is Common Core (US) the only board for Phase 1, or do we launch CBSE/ICSE pages simultaneously?
  6. Grades in scope: K–8 only, or do we stub out 9–12 with "coming soon"?
